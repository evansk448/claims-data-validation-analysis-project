# Claims Data Validation & Analysis Project

## Overview
This project uses SQL to analyze and validate a mock insurance claims database. The goal is to demonstrate how SQL can be used for both business reporting and data quality testing.

## Objectives
- Analyze claims trends and payment activity
- Identify missing, duplicate, or inconsistent data
- Validate business rules related to claims, payments, and subrogation
- Document findings in a clear, business-focused format

## Skills Demonstrated
- SQL querying
- Data validation
- Data quality analysis
- Business rule validation
- Claims workflow understanding
- QA/data testing mindset
